# Risk Engine Integration API

Данный документ описывает формат взаимодействия между Торговым ботом и Risk Engine (RE).

---

## 1. Схема взаимодействия (Approval Flow) — SHADOW MODE ACTIVE 🛠️

**ВНИМАНИЕ:** В данный момент бот работает в «Shadow Mode». Это значит:
1. Бот отправляет запрос в RE.
2. Бот **сразу** открывает сделку по сигналу (не дожидаясь ответа).
3. RE присылает ответ (approve/reject), бот просто логирует его в БД для накопления статистики.

---


## 2. Формат запроса (Торговый бот → RE)

Запрос отправляется как текстовое сообщение в Telegram.

### Поля сообщения:
*   **hash**: Уникальный ID сигнала (обязательно вернуть в ответе).
*   **Symbol/Side/Entry/TP/SL**: Базовые параметры сделки.
*   **RiskReward**: (Опционально) R:R от Midas.
*   **Probability**: (Опционально) Вероятность успеха от Midas.
*   **WinRate**: (Опционально) Исторический WinRate.
*   **Trend**: `bullish` или `bearish`.
*   **--- Raw Signal ---**: Полный оригинальный текст сигнала.

**Пример:**
```text
🔔 Request for approval
hash: abc123def456

Symbol: ETHUSDT
Side: Buy (LONG)
Entry: 2450.5
TP1: 2500 | TP2: 2550 | TP3: 2600
SL: 2400

RiskReward: 7.8
Probability: 89.0
Trend: bullish

--- Raw Signal ---
ETHUSDT.P 60M - 🟢 Strong BUY
... (оригинальный текст) ...
```

---

## 3. Формат ответа (RE → Торговый бот)

RE должен отправить `POST` запрос на Webhook бота.

**Endpoint:** `https://midas-trade.mooo.com/api/re/callback`
**Content-Type:** `application/json`

### JSON Body:
```json
{
    "hash": "abc123def456",
    "decision": "approve", 
    "score": 0.85,
    "reason": "Trend confirmed, VaR OK",
    "secret": "123QWEasd"
}
```

| Поле | Тип | Обязательно | Описание |
| :--- | :--- | :--- | :--- |
| **hash** | string | Да | Тот же хэш, что был в запросе. |
| **decision** | string | Да | `"approve"` (одобрить) или `"reject"` (отклонить). |
| **score** | float | Нет | **Динамический сайзинг.** Бот умножит объем входа на этот коэффициент (например, 0.5 = вход на 50% лота). |
| **reason** | string | Нет | Пояснение (пишется в логи бота). |
| **secret** | string | Да | Токен авторизации: `123QWEasd` |

---

## 4. Тайминги и Безопасность

*   **Таймаут:** Бот ждет ответа **5 минут**. После этого сделка автоматически отменяется (`timeout`).
*   **Токен:** Поле `secret` должно в точности совпадать с настройкой в `.env` бота.
*   **Conflict:** Убедитесь, что мост (estafetabot) не использует `polling` вместе с другими экземплярами, иначе сообщения могут теряться.
