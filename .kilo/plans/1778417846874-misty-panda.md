## Фаза 1: Bybit API Smoke Test (без Telegram, без Docker)

**Цель**: убедиться, что Bybit-ключи работают, баланс читается, ордера выставляются.

### 1.1. Тест баланса

Создать скрипт `scripts/smoke_test_bybit.py`:

```python
"""Bybit API smoke test — balance + test order."""
from pybit.unified_trading import HTTP
import os
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("BYBIT_API_KEY")
API_SECRET = os.getenv("BYBIT_API_SECRET")
TESTNET = os.getenv("BYBIT_TESTNET", "false").lower() == "true"

def test_balance():
    client = HTTP(testnet=TESTNET, api_key=API_KEY, api_secret=API_SECRET)
    resp = client.get_wallet_balance(accountType="UNIFIED", coin="USDT")
    print(f"retCode={resp['retCode']}, msg={resp.get('retMsg')}")
    if resp["retCode"] == 0:
        coins = resp["result"]["list"][0]["coin"]
        for c in coins:
            print(f"  {c['coin']}: equity={c.get('equity')}, wallet={c.get('walletBalance')}")
    else:
        print(f"ERROR: {resp}")

def test_place_order():
    client = HTTP(testnet=TESTNET, api_key=API_KEY, api_secret=API_SECRET)
    # Testnet: place minimum order
    resp = client.place_order(
        category="linear",
        symbol="BTCUSDT",
        side="Buy",
        orderType="Market",
        qty="0.001",
        isLeverage=1,
        positionIdx=0,
        reduceOnly=False,
    )
    print(f"ORDER retCode={resp['retCode']}, msg={resp.get('retMsg')}")
    if resp["retCode"] == 0:
        print(f"  orderId={resp['result']['orderId']}")

if __name__ == "__main__":
    test_balance()
    # test_place_order()  # раскомментировать для теста ордера
```

**Запуск**:
```bash
cd midas-trading-bot
python scripts/smoke_test_bybit.py
```

**⚠️ ВАЖНО**:
- Сначала проверить баланс — без ордера.
- Для теста ордера **обязательно** `BYBIT_TESTNET=true` в `.env`.
- Real trading: `BYBIT_TESTNET=false` — ордер на $0.001 BTC ≈ $80.

### 1.3. ⚠️ Bybit Proxy — обход геоблокировки

`api.bybit.com` заблокирован из РФ. **Dumalka/REDU** ходит через  
`BYBIT_PROXY_URL=http://100.117.168.63:8002` (Tailscale IP, отдельный VPS-сервис).

**Midas-бот в `bybit_client.py` (pybit) подключается к `api.bybit.com` напрямую — без прокси.**  
Варианты для деплоя:

| Вариант | Где работает | Что нужно |
|---------|-------------|-----------|
| **A. VPS за пределами РФ** | api.bybit.com доступен | Ничего — pybit работает напрямую |
| **B. VPS в РФ + Tailscale** | bybit-proxy на 100.117.168.63:8002 | Добавить `HTTP_PROXY` в docker-compose или поддержку прокси в `bybit_client.py` |
| **C. VPS в РФ + свой прокси** | Любой VPS с выходом в интернет | Поднять nginx-прокси или SOCKS5 |

**Для варианта B/C**: добавить в `docker-compose.yml` → `bot.environment`:

```yaml
environment:
  HTTP_PROXY: http://100.117.168.63:8002
  HTTPS_PROXY: http://100.117.168.63:8002
```

Либо в `.env`: `HTTP_PROXY=http://100.117.168.63:8002`.

### 2.2. Что происходит при такой конфигурации

| Компонент | Переменная | Поведение |
|-----------|-----------|----------|
| **Risk Engine** | `RE_URL=""`, `RISK_ENGINE_ENABLED=true` | RE не вызывается (URL пустой) → `notify_risk_engine()` возвращается сразу. Торговля разрешена (gating check пройден) |
| **Dumalka** | `DUMALKA_MODE=off` | Команды Dumalka отклоняются (HTTP 400). Бот САМ управляет SL/TP: при TP2 → SL в безубыток, TP3/SL нативно на Bybit |
| **LLM** | `LLM_VALIDATION_ENABLED=false`, `LLM_ENABLED=false` | OpenRouter не вызывается. Сигналы парсятся только regex-парсером |
| **RE callback** | `RE_URL=""` | События (open, tp1_hit, sl_hit) не отправляются → некому читать, нет смысла |
| **Telegram** | Без изменений | Сигналы принимаются, уведомления отправляются |

**Ключевой фолбэк (telegram_listener.py:405-563)**:
- Если `RISK_ENGINE_ENABLED=true` и `RE_URL` не пуст, но RE не отвечает (timeout/error) → бот открывает сделку с 30% размером (`re_score=0.3`).
- Если `RE_URL=""` → httpx вызовет `InvalidURL` → попадаем в `except` → фолбэк 30% → сделка открывается.

## Tailscale: подключение к Risk Engine/Dumalka

RE и Dumalka живут на отдельной VM с Titan V GPU. Связь через VPN-сеть
**Tailscale** (IP `100.87.26.107` в оригинальном `.env`).

Порядок включения после установки Tailscale:

```bash
# 1. Установить Tailscale на VPS с ботом
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up

# 2. Проверить IP RE-сервера в Tailscale-сети
tailscale status
# Найти RE-машину (имя или 100.x.y.z)

# 3. Установить RE_URL в .env на Tailscale IP
# RE_URL=http://100.87.26.107:8000

# 4. Если нужна Думалка — переключить режим
# DUMALKA_MODE=active
# DUMALKA_TOKEN=dumalka_secret_2026

# 5. Перезапустить бота
docker compose restart bot
```
